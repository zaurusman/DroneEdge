Session Naming — When does it happen?

The recording pipeline starts immediately on ARM. When should the user be able to set a name?

A

Name before recording starts

Tap REC → dialog appears with date/time pre-filled → user edits or accepts → recording starts.

Pro

  • Name is set from the start

Con

  • Adds friction — you might miss the moment
B

Name after recording stops

Recording starts immediately → user taps STOP → dialog appears pre-filled with date/time → user edits or accepts.

Pro

  • Zero friction before capture; name in context of what just happened

Con

  • One extra step after every session
C

Auto-name always, rename from gallery

Recording always saves as date/time. User can tap a name in the gallery to rename it later.

Pro

  • No interruption to the recording flow at all

Con

  • Easy to forget to rename; disconnected from the session
